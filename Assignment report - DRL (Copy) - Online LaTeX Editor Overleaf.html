Report of Assignment 2 : Deep Q-learning
Niccol`o Maccarini 1 Ambar Qadeer (s2536307) 1
1. Introduction
Tabular reinforcement learning algorithms are an efficient
and accessible tool to solve many classic reinforcement
learning problems, however, with an increase in the state/
action space of the environment, these methods quickly
become infeasible. For a continuous or large state space,
storing Q-values corresponding to each state and action is
often atleast inefficient, if not impossible — enter Deep Q-
learning Networks (DQNs). DQNs employ a neural network
(NN) to approximate a function that returns the correct Q-
values given a state. This simple idea does away with the
need for storing and updating large tables, however, any
attempt at approximating the Q-value function introduces
its own challenges and complexities, namely:
• The deadly triad :
1. Coverage
2. Correlation
3. Convergence
In the following report, we discuss these problems as we de-
scribe our attempt to create an efficient DQN agent to solve
the classic cartpole problem and attempt to tackle the chal-
lenges it introduces. Throughout the study, we continue to
rely on the helpful python code, lecture material, suggested
literature and guidance provided by the Leiden University,
”Reinforcement Learning” course team, organizers, teaching
assistants and our teachers (1; 2; 3).
2. Network Structure
The first step is to build a Deep Q-network that will com-
pute the Q-values of every state. Since each state of the
environment consists in a four-dimensional real tensor, we
don’t need to use convolutional layers, instead we opted for
a standard architecture with three hidden layers with twenty-
four nodes each. The action spaces consists of two elements,
therefore the output layer of the network will be a tensor
with two elements, the maximum of which corresponding to
the optimal action given the input state. The hidden layers
are activated by the Relu function, while the output layer
is activated by the identity. After a couple of tests we also
decided to initialize all weights with HeUniform, as the
resulting performance ended up being slightly better than
using weights initialized with Glorot. Before settling with
this model several other options were tried: hidden layers
with up to 100 nodes were tested, with no significant im-
provement in the model’s performance. Similarly, we tried
to remove a hidden layer, registering a slightly worse and
less stable performance.

3. Exploration strategy and experience replay
The convergence proofs of tabular methods rely heavily on
eventually traversing the entire state space. This is exactly
what we attempt to avoid with a DQN to make our agent
more efficient. We tackle this by adding an exploratory
element to our search for the ideal NN weights. The rein-
forcement learning algorithm that we first tried followed
the pseudocode of algorithm 1: we fix a number M of runs,
each with a maximum T of steps. At each step we select an
action, and access the environment to get the reward and the
next state, then we define the sum of squares of the differ-
ences between the moving targets and the images Q(st, at),
performing a gradient descent step on this quantity.

Initialize action-value function Q with random weights
for i = 1 to M do
Initialize first state s0
for t = 1 to T do
Select an action at
Access the environment and execute at
Save observed reward rt and next state st+1
Set st = st+1 and targets:
for st visited state do
yt =
{
rt if st+1 is terminal
rt + γ maxa′ Q(st+1, a′) otherwise
end for
Set sum sq = ∑(yt − Q(st, at))2
Perform a gradient descent step on sum sq
if then
st is terminal break
end for
end for
There are different ways with which, given a state, an ac-
tion can be selected, first of all we tried out an ε-greedy
policy with different values of ε. A couple of facts need
to be taken into account: the cartpole environment assigns
only positive rewards to each state, and, on average, at the
very first epochs the algorithm can keep the pole straight for
8-10 steps before it falls. Recall that weights initialized via
HeUniform are on average zero, this implies that before the
first run the Q-values will be on average zero (as they can be
written as linear combination of the weigths), while after the
first run the Q-values of the visited positions will be on av-
erage positive (the first around 10, the second around 9, and
so on). It follows that following a greedy policy the algo-
rithm will not be incentivised to deviate from the first steps,
even if scarcely satisfying. Hence it’s not surprising that the
best values for ε are high, since higher rates of exploration
help the algorithm to find out which actions result in longer
runs. Indeed, a variant of standard ε-greedy exploration that
yielded good performances consists in starting with ε = 1,
and then decaying this value after every run. A good factor
of decay was found to be 0.95. In this way during the first
runs actions are selected completely randomly, therefore
when ε decays below a certain threshold we already have
raw estimates of the Q-values, and as a consequence con-
vergence is faster than other methods. Despite this, results
yielded by this method turned out to be unsatisfying, not
reaching a good performance even for high values of M and
T . The reason for this might lay in the fact that fitting the
model at every iteration using the full batch of previously
visited states will lead to an overtraining of the network on
already seen data, hence it will have poor generalization
properties and heavy bias. A way to face this problem could
be to switch to an online update rule, summarised in pseu-
docode 2. Now at every step we compute only the target yt
for the last visited state, and perform a gradient descend step
on the squared difference of yt with Q(st, at). This method
proved to be much more efficient, reaching maximums of
200 consecutive steps without falling, unfortunately it was
also extremely unstable: it was unlikely that after a couple
of runs with high number of consecutive steps the algorithm
could keep or beat the record, but the score systematically
dropped. As a consequence, averaging the score (i.e. the
number of consecutive steps in which the pole doesn’t fall)
per run over eight or ten repetitions of the algorithm, will
result in a flat line, with little to no average improvement.

end for

of running the gradient descent step, namely initializing a
replay buffer and sampling at each step a batch of states
from the memory, fitting the model with these data. As
predicted, this method performed much better than standard
deep Q-learning, but it was also much slower, to speed it up
we opted to make updates less frequent, namely every four
steps. The procedure is summarised in algorithm 3.
